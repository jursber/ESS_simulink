## 1. 项目概述

### 1.1 核心目标

构建一个储能运营模拟计算器，帮助用户理解**所有外部条件如何传导到储能收益**。

用户通过滑块、自定义参数、数据选择等方式创建不同方案，进行单方案分析和多方案对比，工具智能分析方案差异原因。

### 1.2 业务角色

| 角色 | 简称 | 说明 |
|------|------|------|
| 售电公司 | Retailer | 从电力市场购电，向终端用户售电 |
| 储能投资商/运营商 | ESS Operator | 投资并运营储能系统，同一概念 |
| 终端用户 | User | 用电方，与售电公司有合同关系 |

### 1.3 范围约定

- v1.0.0 仅含**电能量电费**，不含输配电价、线损、系统运行费、政府基金、需量电费
- v1.0.0 不含光伏
- 当前仅有河南（henan）地区数据（mock）
- 假设售电公司代理一个终端用户（架构预留多家用户的扩展能力）

## 2. 物理模型

### 2.1 功率/电量关系

所有计算为小时级，1 kW × 1 h = 1 kWh，功率数据直接视为电量（kWh）。

### 2.2 关口表关系

```
load_real[t] = load_grid[t] + load_ESS[t]
```

| 变量 | 含义 | 单位 | 说明 |
|------|------|------|------|
| load_real | 终端用户设备实际负荷 | kWh | 用户电费唯一结算依据 |
| load_grid | 关口表读数，电网侧计量 | kWh | 售电公司购电结算量 |
| load_ESS | 储能充放电电量 | kWh | **充电为负，放电为正** |

### 2.3 SOC 递推

```
SOC[t+1] = SOC[t] + ΔSOC[t]

其中：
  充电时（load_ESS[t] < 0）：
    ΔSOC[t] = |load_ESS[t]| × η_single / Cap_rated

  放电时（load_ESS[t] > 0）：
    ΔSOC[t] = −load_ESS[t] / η_single / Cap_rated

  η_single = sqrt(η_roundtrip)
  η_roundtrip ≈ 85%，η_single ≈ 92.2%
```

约束：
- `0 ≤ SOC[t] ≤ 1`（实际使用 SOC_min ~ SOC_max，默认 10%~90%）
- 充放电不可同时（同一小时不能既充电又放电）
- 跨日 SOC 连续传递

### 2.4 功率约束

```
|load_ESS[t]| ≤ P_max
P_max = Cap_rated × C_rate          （C_rate 默认 0.5）
```

### 2.5 容量衰减

```
Cap_eff(N) = Cap_rated × (1 − r_degrade)^N
r_degrade 默认 2.5%/年
```

## 3. 电价模式

以下为售电公司→终端用户的**售电端**电价模式。

### M1：行政分时电价（无售电公司）

电网公司直接设定峰/平/谷时段及价格。用户直接向电网缴费。

```
P_user[t] = TOU_table(period[t])
```

### M2：行政分时电价（"江苏模式"，标记"即将淘汰"）

售电公司给用户一个**月度平段基础电价** P_base。时段划分与行政分时电价一致（峰/平/谷三段），但价格由 P_base × 系数生成。

```
P_user[t] = P_base × coefficient[period[t]]
其中：period[t] 沿用行政分时电价的时段划分
     系数_峰 = 1.6，系数_谷 = 0.4，系数_平 = 1.0
```

用户向电网缴纳电能量电费。电网将电能量电费转付售电公司。

### M3：合同分时电价

售电公司与用户自行约定峰/平/谷时段及价格。

```
P_user[t] = Contract_TOU_table(period[t])
```

时段结构与 M1 相同（峰/平/谷三段），但具体时段划分和价格均独立设定。

### M4：现货联动电价

不同天同一时刻价格相同，等于全月该小时日前价格的算术平均。

```
P_user[h] = mean_{d in month}(P_da[d, h])
其中 h = 0..23，对给定 h，该月所有天的价格相同
```

### M5：售电公司一口价

```
P_user[t] = P_flat          （24 小时统一价格）
```

## 4. 商业模式与优化目标

"XX 收益最高"指以该目标优化储能调度曲线。不同优化目标导致不同的充放电决策。

### B1：用户 + 储能投资商（无售电公司）

- 适用电价模式：M1（行政分时电价）
- 用户向电网交分时电费，同时向储能投资商支付储能服务费
- 自动以 P_eff = P_TOU 进行峰谷套利优化（无需选择优化目标）

### B2：三方独立（售电 / 储能 / 用户各自独立）

| 子模式 | 优化目标 |
|--------|----------|
| B2a | 售电公司收益最优 |
| B2b | 储能投资商收益最优 |
| B2c | 终端用户收益最优 |

> **警示**：B2a 模式下售电公司有动机在 P_user ≫ P_rt 时储能充电（抬高 load_grid 赚取价差），导致用户电费大幅上升。这在经济上是"正确"的——售电公司利用储能最大化自身利润而损害用户利益。现实监管中通常不允许售电公司单方面控制用户侧储能调度。

### B3：储售一体 vs 用户独立

售电公司和储能投资商为同一主体（内部无需转移支付）。

| 子模式 | 优化目标 |
|--------|----------|
| B3a | 储售一体总收益最高 |
| B3b | 终端用户收益最高 |

### B4：三方一体

三方为同一主体，**总体收益最高**。

## 5. 公式体系

### 5.1 符号约定

| 符号 | 含义 | 单位 |
|------|------|------|
| P_user[t] | 时刻 t 的用户侧电价（取决于电价模式 M1~M5） | 元/kWh |
| P_da[t] | 时刻 t 的用户侧统一结算点日前价格 | 元/kWh |
| P_rt[t] | 时刻 t 的用户侧统一结算点实时价格 | 元/kWh |
| P_合约[t] | 时刻 t 的中长期合约电价 | 元/kWh |
| Q_合约[t] | 时刻 t 的中长期合约净电量 | kWh |
| Q_日前[t] | 时刻 t 的日前市场出清电量 | kWh |
| load_real[t] | 时刻 t 的用户实际负荷 | kWh |
| load_grid[t] | 时刻 t 的关口表读数 | kWh |
| load_ESS[t] | 时刻 t 的储能充放电（充电负/放电正） | kWh |
| r_user | 用户分成比例 | — |
| Cap_rated | 储能额定容量 | kWh |
| UnitCost | 建设单价 | 元/kWh |

### 5.2 用户侧电费

#### 无储能

```
用户电费_无储能 = Σ_t (load_real[t] × P_user[t])
```

#### 有储能

```
用户电费_有储能 = Σ_t (load_grid[t] × P_user[t])
注意：结算量是 load_grid，不是 load_real
```

#### 电费节省

```
电费节省 = 用户电费_无储能 − 用户电费_有储能
```

#### 用户净收益 & 支付给运营商

```
用户净收益    = 电费节省 × r_user
运营商收入   = 电费节省 × (1 − r_user)
```

> 注：用户已在缴纳电费时实际支付了全额，"电费节省"是相对无储能情景的减少额。用户从节省额中以现金方式按 `(1 − r_user)` 比例支付给运营商。

### 5.3 售电公司侧

#### 售电收入

```
售电收入 = Σ_t (load_grid[t] × P_user[t]) = 用户电费_有储能
```

#### 购电成本

```
C_购电 = C_中长期 + C_日前 + C_实时

C_中长期 = Σ_t (Q_合约[t] × P_合约[t])

C_日前   = Σ_t [(Q_日前[t] − Q_合约[t]) × P_da[t]]

C_实时   = Σ_t [(load_grid[t] − Q_日前[t]) × P_rt[t]]
```

#### 储能对购电成本的影响

Q_合约 和 Q_日前 是预定量，不受储能影响。储能仅改变 load_grid：

```
ΔC_购电 = Σ_t [(load_grid_with_ESS[t] − load_grid_without_ESS[t]) × P_rt[t]]
        = Σ_t [(−load_ESS[t]) × P_rt[t]]
```

#### 售电基础收益

```
售电基础收益 = 售电收入 − C_购电
```

### 5.4 储能投资商侧

```
运营商服务收入 = 电费节省 × (1 − r_user)
年运维成本     = 储能初始投资 × r_om
               = Cap_rated × UnitCost × r_om         （r_om 默认 1%）
年净现金流     = 运营商服务收入 − 年运维成本
```

### 5.5 组合体公式

#### 储售一体（B3a）

```
一体总收益 = 售电基础收益 + 运营商服务收入
           = (售电收入 − C_购电) + 电费节省 × (1 − r_user)
```

#### 三方一体（B4）

```
总收益 = 用户净收益 + 售电基础收益 + 运营商服务收入
       = 电费节省 × r_user + (售电收入 − C_购电) + 电费节省 × (1 − r_user)
       = 电费节省 + 售电收入 − C_购电
       = (用户电费_无储能 − 用户电费_有储能) + 售电收入 − C_购电
```

### 5.6 投资评价

```
总投资         = Cap_rated × UnitCost           （UnitCost 默认 0.9 元/Wh = 900 元/kWh）
静态回收期     = 总投资 / 年净现金流            （年）
NPV            = −总投资 + Σ_t (净现金流_t / (1 + r_discount)^t)
IRR            = 使 NPV = 0 的折现率            （迭代求解）
等效循环次数   = Σ |放电量| / Cap_rated
```

### 5.7 统一优化框架：有效价格信号

所有商业模式通过对"有效价格信号 P_eff"最大化 Σ(load_ESS[t] × P_eff[t]) 统一求解：

| 商业模式 | P_eff 推导 | 直觉解释 |
|----------|-----------|----------|
| B1（用户+储能） | P_eff = P_TOU | 谷充峰放套利 |
| B2a（售电最优） | P_eff = P_rt − P_user | 售电公司赚取 P_user 与 P_rt 价差 |
| B2b（储能最优） | P_eff = P_user | 最大化用户电费节省 = 最大化运营商收入 |
| B2c（用户最优） | P_eff = P_user | 最大化用户电费节省 |
| B3a（储售一体） | P_eff = P_rt − r_user × P_user | 组合体同时赚取购销价差和储能分成 |
| B3b（用户最优） | P_eff = P_user | 最大化用户电费节省 |
| B4（三方一体） | P_eff = P_rt | 最大化社会总福利 = 最小化实时购电成本 |

> 推导过程见 `src/core/dispatch.py:compute_effective_price()`。关键洞察：P_user 相关项在不同优化目标中充当"内部转移支付"，B4 中全部抵消，仅余 P_rt。

## 7. 算法需求

### 7.1 核心任务

根据以下输入，计算储能**每小时最优充放电电量** load_ESS[t]（t = 0..23）：

- 电价模式（M1~M5）
- 商业模式与优化目标（B1~B4）
- 储能参数（容量、效率、SOC 约束等）
- load_real 曲线
- 电价数据（P_user, P_da, P_rt, etc.）

### 7.2 输出

- 24 小时 load_ESS[t] 序列（kWh）
- 对应 SOC[t] 序列
- 各角色收益分解
- 关键指标（IRR / NPV / 回收期 / 等效循环次数）

### 7.3 约束

- SOC 边界（SOC_min ≤ SOC[t] ≤ SOC_max，跨日连续）
- 功率约束（|load_ESS[t]| ≤ P_max）
- 充放电互斥
- 日初 SOC = 前日日末 SOC

### 7.4 待下一轮需求讨论

- 具体优化建模方式（线性规划 / 动态规划 / 启发式）
- 日前市场套利调度 vs 合同电价调度的策略差异
- 多目标优化时的权重或约束处理

## 8. 架构原则

1. **后端忠实建模**：按真实市场规则计算，不做简化近似。即使当前售电公司只有一个用户且电量 1:1 联动，后端各环节独立计算
2. **前端灵活聚合**：展示层可做合并，但不影响后端计算逻辑
3. **参数全可调**：所有储能参数、电价参数、合约参数均可在 Web UI 中调整
4. **方案化管理**：用户可保存配置为方案，创建多方案并对比
5. **可扩展**：数据结构预留多用户、多地区、光伏接入等未来扩展能力
6. **数据与代码分离**：电价表、合约数据等以独立文件存储，方便替换

### 9.2 方案管理

**方案 = 全局默认参数 + 方案私有参数覆盖**

#### 创建方案
- 新建方案时，自动加载全局默认参数
- 用户可从全局参数库中**选择性添加**参数到当前方案，成为该方案的私有参数
- 未被添加的参数沿用全局默认值

#### 方案对比
- 新建方案二时，同样默认加载全局缺省参数（不继承方案一的私有参数）
- 方案二同样可独立添加/修改自己的私有参数
- 选择任意两个或多个方案进行并排对比

#### 复制参数
- 任意方案可一键复制后台已保存的另一个方案的**全部或部分**私有参数
- 复制操作将目标方案的参数值覆盖到当前方案的对应参数

### 9.3 结果展示

- 单方案：收益分解瀑布图、SOC 曲线、充放电调度图、投资指标表
- 多方案对比：并排对比表、差异分析（自动分析差异原因）
- 数据导出
