## 第 2 轮（需求讨论，2026-05-16）

- **R015**：项目功能模式——用户通过滑块、自定义参数、数据选择等方式配置场景，支持单方案分析和多方案对比，工具智能分析方案差异原因
- **R016**：三个业务角色——售电公司、储能投资商（=运营商）、终端用户
- **R017**：物理计量关系——load_grid = load_real - load_ESS，load_real 是终端用户唯一结算依据
- **R018**：储能充放电符号约定——充电为正、放电为负（待确认与导入公式的一致性）
- **R019**：电价模式1——行政分时电价（无售电）：电网公司设定的峰谷平电价，用户直接向电网缴费
- **R020**：电价模式2——行政分时电价（带售电，江苏模式，标记"即将淘汰"）：售电公司给月度不分时平段电价 P_flat，峰=1.6×P_flat，谷=0.4×P_flat，平=P_flat；用户另向电网缴纳输配电价等
- **R021**：电价模式3——合同分时电价：售电公司与用户约定固定峰谷平电价
- **R022**：电价模式4——现货联动电价：每小时电价不同，但同一天不同小时对应 = 售电公司当月日前现货算术平均电价
- **R023**：电价模式5——售电公司一口价：24小时统一固定电价
- **R024**：商业模式1——仅用户+储能投资商（无售电），行政分时电价，用户向电网交分时电费 + 向运营商支付"返还费用"
- **R025**：商业模式2——三方独立，分别优化各自收益（2a:售电最优/2b:储能最优/2c:用户最优）
- **R026**：商业模式3——储售一体 vs 用户独立（3a:储售一体收益最高/3b:用户收益最高）
- **R027**：商业模式4——三方一体，总体收益最高
- **R028**：关键公式——储能收益 = Σ(储能各小时充放电量 × 各小时电价)
- **R029**：关键公式——用户返还给运营商的费用 = 储能收益 × (1 - 用户侧分成比例)
- **R030**：需人工录入——售电合同模式及参数、中长期合约（量/价）、日前报电量数据、储能参数
- **R031**：算法核心——根据商业模式计算储能最优运行策略（每小时充放电量）
- **R032**：售电公司购电成本 = 中长期电费 + 日前电费 + 实时电费
- **R033**：本次不考虑需量电费和光伏
- **R034**：假设售电公司只有一个用户（终端用户），用户用电量 = 售电公司实际购电量
- **R035**：SOC 约束——0%~100%，充放电不能同时进行，跨日连续传递
- **R036**：储能往返效率约 85%，单程效率 = sqrt(往返效率)
- **R037**：储能功率约束——最大充放电功率通常 = 额定容量 × 0.5C
- **R038**：等效满充满放次数 = 总放电量 / 额定容量
- **R039**：投资收益指标——IRR、NPV、静态投资回收期
- **R040**：建设单价 700~1200 元/kWh，磷酸铁锂设计寿命 10 年
- **R041**：敏感性分析变量——建设单价、用户分成比例、合同价差、日前价差、调度策略、运维比例、项目寿命
- **R042**：容量衰减——第N年有效容量 = 初始容量 × (1 - 年衰减率)^N，年衰减率通常 2%~3%
- **R043**：月度批发合同（中长期+日前+实时），售电公司从中长期和现货市场组合购电
- **R044**：江苏模式中，用户额外向电网缴纳输配电价、线损费用、系统运行费、政府性基金及附加（按用电量×核定单价）
- **R045**："运营商"、"储能投资商"等描述统一指代储能投资商（ESS投资者/运营商）

## 第 3 轮（需求澄清，2026-05-16）

- **R046**：储能符号约定确认——充电为负、放电为正，storage = load_ESS
- **R047**：CSV 原始列含义确认——load = load_grid（关口表读数），storage = load_ESS（储能功率）
- **R048**：Load_real = load + storage（需修正已处理数据！此前用 load - storage 是错的）
- **R049**：Q2确认——现货联动电价 = 各小时全月 day_ahead 算数平均，每天同小时价格相同
- **R050**：用户电费节省 = 无储能时的电费 - 有储能时的电费；用户支付给运营商的费用 = 电费节省 × (1 - 用户分成比例)
- **R051**：售电公司购电量 = load_grid（关口表读数），而非 load_real
- **R052**：商业模式3"储售一体"= 售电公司和储能投资商是同一主体，无需内部转移
- **R053**："XX收益最高"指以该目标优化储能调度曲线，不同优化目标导致不同的充放电决策
- **R054**："用户保留储能收益 × 用户分成比例"表述不准确，实际是用户电费账单减少额中，运营商拿走 (1-分成比例)，用户净留存 分成比例
- **R055**：合同电价和现货购电成本可能存在价差矛盾（如0:00合同100但现货200），不同优化目标会产生相反的调度决策

## 第 5 轮（售电公司购电成本，2026-05-16）

- **R068**：售电公司购电成本 C = C_中长期 + C_日前 + C_实时
- **R069**：C_中长期 = Σ(Q_合约,t × P_合约,t)，中长期合约包括年度双边、月度集中竞价等
- **R070**：C_日前 = Σ[(Q_日前,t - Q_合约,t) × P_日前,t]，量差结算——日前出清电量与合约电量的偏差按日前价格结算
- **R071**：C_实时 = Σ[(Q_实际,t - Q_日前,t) × P_实时,t]，Q_实际,t = load_grid[t]
- **R072**：购电成本中仅 Q_实际,t（=load_grid）受储能运行影响，Q_合约和 Q_日前为预定量
- **R073**：储能对购电成本的影响仅通过实时偏差电费传导：ΔC = Σ[(-load_ESS[t]) × P_实时,t]
- **R074**：需 mock 数据：Q_合约,t、P_合约,t、Q_日前,t（三组小时级时间序列）
- **R075**：后端严格按真实市场规则计算，不做简化或近似——即使当前售电公司只有一个用户、用户电量与购电电量 1:1 联动，后端也独立计算各环节
- **R076**：架构需支持未来扩展（如一家售电公司下挂多个用户），前端可做展示层合并，后端不能偷工减料

## 第 6 轮（PRD 修正 & 参数管理体系，2026-05-16）

- **R077**：运营商年运维成本 = 储能初始投资 × 1%
- **R078**：储能初始投资单价 = 0.9 元/Wh（= 900 元/kWh）
- **R079**：所有涉及参数的项目都应放在 Web 初始参数设置页面中
- **R080**：全局初始参数页面包含全部可调节参数，作为系统参数库
- **R081**：方案 = 全局默认参数 + 方案私有参数覆盖；新建方案自动加载全局默认参数
- **R082**：用户可从全局参数库选择性添加参数到方案中作为私有参数，未添加的沿用全局默认值
- **R083**：新建对比方案（方案二）默认加载全局缺省参数，不继承方案一的私有参数
- **R084**：任意方案可一键复制其他已保存方案的全部或部分私有参数
