# 2026-06-08 夜间大改记录

> 目标：在不频繁打扰业务输入的前提下，把当前系统从“单方案页面较完整，其余页面占位”推进到“多页面可用、后端接口可测、数据资产可见”的工程状态。

## 已完成

### 单方案分析

- 方案槽位从 A/B/C 扩展为 A/B/C/D，匹配“最多 4 个方案对比”的产品目标。
- 底部“比较”不再打开空弹窗，而是把已记录槽位带入“多方案对比”页。
- 右侧参数区的“编辑”入口不再弹占位提示，而是跳转到缺省参数对应面板。

### 多方案对比

- 新增真实页面结构：方案列、刷新对比、指标矩阵、关键 KPI 卡片。
- 后端新增 `POST /api/compare`，最多支持 4 个方案。
- 当前指标覆盖：
  - 总社会福利提升
  - 终端用户节费
  - 用户实际用电成本
  - 储能 IRR
  - 储能静态回收期
  - 储能年套利创值
  - 日等效循环次数
  - 售电公司利润
  - 光伏 IRR
  - 光伏年发电量
  - 光伏本地消纳率
  - 用户负荷变异系数
- 指标矩阵支持按指标方向高亮更优值。

### 方案管理

- 新增真实方案管理表格。
- 后端新增方案 CRUD：
  - `POST /api/scenarios`
  - `PUT /api/scenarios/{scenario_id}`
  - `POST /api/scenarios/{scenario_id}/duplicate`
  - `DELETE /api/scenarios/{scenario_id}`
- 前端支持新建、改名、复制、加载、删除。

### 模型管理

- 新增模型管理页，按类别列出当前已接入或简化接入的模型：
  - 调度模型
  - 零售电价模型
  - 商业模式/优化目标
  - 批发侧结算模型
  - 投资评价模型
- 后端新增 `GET /api/models`。
- 调度模型、结算模型、投资模型提供超参草稿编辑入口。
- 后端新增 `PUT /api/models/{model_id}/params`，当前先保存到 `data/model_params/{model_id}.json`，后续再接入真实计算。
- 模型目录会读取已保存的超参草稿并回显，模型卡片显示“草稿已保存”。

### 缺省参数

- 新增“数据资产总览”面板。
- 后端新增 `GET /api/data-assets`，按业务类别扫描 `data/`：
  - 全局参数
  - 分时电价
  - 现货价格
  - 统调负荷
  - 用户典型负荷
  - 光伏出力
  - 交易策略持仓
  - 需量/容量单价
  - 数据可信度
- 页面展示文件数量、路径、字段预览、行数、KB 和可信度登记。

### 文档系统

- 新增帮助文档页。
- 后端新增：
  - `GET /api/docs`
  - `GET /api/docs/{doc_id}`
- 左侧文档列表 + 搜索，右侧正文阅读。
- 内置了系统总览、单方案分析、多方案对比、数据可信度说明，并自动纳入 `docs/*.md` 和 `PRD/*.md`。

## 测试

  - 多方案对比 API
  - 方案 CRUD API
  - 模型目录 API

### 底层闭环修复

- `ScenarioConfig.ess_params`、`financial_params` 和 `private_overrides` 已进入 `calculate()` 运行时计算。
- `/api/calculate` 与 `/api/compare` 的储能概览口径同步使用解析后的 ESS 参数。
- API 计算缓存 key 已加入方案内容指纹和 `data/params/*.csv` 文件版本，避免参数调整后返回陈旧结果。
- 删除旧的非 `.v4` 前端 JS/CSS 残留文件，避免后续误判真实入口。
- `frontend/index.html` 对当前 `.v4` 前端资源追加夜间版本号，降低浏览器缓存导致的半旧页面风险。

## 当前边界

- 多方案对比当前基于典型日计算结果，不是全年深度精算。
- 模型超参当前保存为草稿，尚未全部接入调度引擎。
- 方案保存当前只保存基础选择项；完整前端参数快照仍需下一轮设计。
- 文档正文是轻量 markdown 渲染，不是完整富文本/知识库系统。
- 剩余已标记 xfail 是江苏模式配置迁移/降级相关，暂不影响河南当前主链路。

## 建议下一步

第一优先级仍然是把“方案 = 全局默认参数 + 前端当前选择 + 方案私有参数覆盖 + 仿真模式”的保存语义彻底定下来。这个语义不稳定，多方案对比和深度精算都会反复返工。
