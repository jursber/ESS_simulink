# ESS_simulink 项目上下文

> 本文把 2026-06-08 至 2026-06-09 的主要会话内容沉淀为项目资料。后续研发、测试和需求讨论以 `D:\Cursor\ESS_simulink` 为项目根目录。

## 项目目标

通过“人工测试驱动需求落地”的方式，提高源网荷储售仿真系统的完整度。当前优先级最高的是把“单方案分析”页面的 Case1 跑通：单储能场景下，按行政分时电价进行调度计算，并能从参数选择、算法运行、结果展示、方案暂存/保存、方案对比形成闭环。

## 工作方法

- 用户负责从产品和业务视角做人工测试，判断交互是否自然、需求是否合理、结果是否符合业务直觉。
- Codex 负责尽量减少用户体力活：监控日志、判断参数变化、验证接口结果、提前发现明显逻辑漏洞、修复前后端和算法链路。
- 测试不是单纯找 bug，而是推动需求成型。每个场景测试都要沉淀为可复用的业务用例、算法入口和验收标准。

## 当前业务主线

### Case1：单储能 + 行政分时电价

目标是完整跑通：

1. 单方案页面所有可选项能正常选择。
2. 系统构成、运行曲线、设备参数、调度目标等字段联动正确。
3. 点击“开始仿真”后有正在计算提示。
4. 后端算法完成后，页面展示正确结果。
5. A/B/C/D 子方案可新增、切换、暂存、保存和比较。

### 后续典型场景

后续测试要逐步覆盖不同算法和业务关系，例如：

- 储能 + 光伏。
- 合同分时 + 售电公司代理。
- 售电公司和储能投资商为同一主体。
- 光伏、用户分别独立。
- 调度权在售电公司。

这些场景涉及不同算法和收益归属，不能只做前端切换，要让后台计算和结果解释同步成立。

## 已形成的产品决策

- 单方案页采用“父方案 + A/B/C/D 子方案”结构。
- 页面首次默认只有 A，A 激活，右侧显示 `+`。
- 点击 `+` 新增下一个子方案，复制当前页面参数快照并激活新子方案。
- 点击 A/B/C/D 是切换和加载子方案，不是保存。
- 任意子方案修改后显示红点，表示未暂存。
- 底部“暂存”只清除当前子方案红点，不等于持久保存父方案。
- 顶部“保存方案”持久保存父方案及其所有子方案。
- 比较按钮比较当前页面内 A/B/C/D 的最新快照，包括未顶部保存的内容。
- 用户净生产负荷永远选中且 disabled。
- 可调负荷本期不实现，只在图中保留禁用元素。
- 光伏/储能取消勾选后，相关容量、设备参数按钮、投资分析 tab 和结果字段要立刻置空或禁用。
- “主流三部制”当前只作为页面文案，底层仍映射到已实现的 `GUANGDONG_STYLE`。
- “多日深度模拟”当前只显示 disabled tab，本期不实现。
- “电价曲线”待后续原型，不抢跑实现。

## 已实现能力概览

- 后端 `ScenarioConfig` 支持旧单方案 JSON，也支持 `variants.A/B/C/D`。
- `/api/calculate` 支持按 `variant_key` 和当前快照计算。
- `/api/compare` 支持按当前子方案快照比较。
- 储能 disabled 时后端容量和功率置 0，避免调度和 IRR 除零。
- 光伏 disabled 时后端不传 PV 参数，PV 投资结果为空。
- 用户净生产负荷曲线已经进入算法输入。
- 光伏地区和曲线类型已经进入算法输入。
- 前端新增 `frontend/js/workbench.v4.js`，负责 A/B/C/D 子方案工作台逻辑。
- 方案管理器已能显示子方案按钮并加载指定子方案。
- 当前服务地址：`http://127.0.0.1:8000/`。

## 当前风险和注意事项

- Windows/Python 环境曾多次出现 `access violation`，尤其与 `tempfile` 或 uvicorn 进程有关。业务断言未必失败，但进程可能异常退出。
- 服务由 watchdog 拉起，偶尔可能出现浏览器还吃旧脚本缓存。遇到页面表现和代码不一致时，优先检查脚本版本号和 `run.server.log`。
- 当前工作区可能有未提交修改，尤其是前端调试修复和 `data/scenarios/c0e7a74a2a2c.json`。提交前必须先确认 diff。
- `各方经济性概览` 的评级规则当前是硬编码阈值，且“售电公司度电收益”的计算口径疑似多乘了 365，需要产品化修正。
- “无敌”这类评级名称不适合作为正式产品文案，建议改成“极优”或“卓越”。

## 当前经济性评级规则

| 评价对象 | 当前指标 | 当前计算口径 | 评级规则 |
|---|---|---|---|
| 终端用户 | 储能节费，万元/MWh | 用户节费 / 储能容量 MWh | `<0 极差`，`0-2 较差`，`2-4 差`，`4-7 达标`，`7-10 优秀`，`>=10 无敌` |
| 光伏投资 | IRR% | `pv_irr * 100` | 空值 `--`，`<=0 极差`，`0-2 较差`，`2-4 差`，`4-8 达标`，`8-12 优秀`，`>12 无敌` |
| 储能投资 | IRR% | `investment.irr_pct` | 同光伏 IRR 规则 |
| 售电公司 | 度电收益，元/MWh | 当前代码为 `售电利润万元 * 10000 / (年放电量MWh * 365)` | `<0 极差`，`0-1 较差`，`1-3 差`，`3-5 达标`，`5-8 优秀`，`>=8 无敌` |

## 下一步建议测试路线

1. 强刷页面，验证右下角 `+` 能新增 B，并自动激活 B。
2. 在 A 中保持单储能 + 行政分时，点击开始仿真，记录 IRR、日循环、用户节费。
3. 新增 B，取消储能和光伏，验证所有光储结果置空或为 0，并确认比较按钮可用。
4. 新增 C，勾选光伏，设置光伏容量和晴天曲线，验证 PV 年发电量和自用率。
5. 对比 A/B/C，检查结果是否符合业务直觉。
6. 点击底部暂存和顶部保存，刷新页面后确认父方案与子方案仍能加载。
7. 开始讨论经济性评级口径和展示文案的产品化调整。
