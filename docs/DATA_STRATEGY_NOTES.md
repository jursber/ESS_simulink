# 数据策略备忘

> 本文记录当前阶段的数据组织共识，方便多电脑开发时通过 Git 同步上下文。最后更新：2026-06-07。

## 项目当前定位

短期目标不是做完整电力市场规则平台，而是建设“可信数据底座 + 光储/售电收益仿真内核”。数据可信度和业务口径优先于数据来源公开、政策规则全量复刻、复杂市场细节。

## region 维度原则

不是所有数据都必须按 `region` 组织。是否带地区维度取决于数据本身是否具有地区差异。

### 必须按 region/province 组织

- 行政分时电价：各省、各月份、各用电类别不同。
- 现货价格：省级市场差异明显，当前只需要日前统一出清价和实时统一出清价。
- 光伏实际出力/典型曲线：按省份、曲线性质区分；后续替换 mock 为真实数据。
- 统调负荷：用于中长期分解曲线，按省份组织；避免称为“系统负荷”。
- 需量/容量单价：与地区、电压等级、用户类别或合同约定相关，应支持地区化，但前端入口后续再定。

### 不需要按 region 组织

- 工厂用户典型负荷曲线：当前设计为 16 种典型工厂负荷特征。相同生产特征的工厂在北京或河南可复用同一负荷曲线，不强制地区化。
- 自定义负荷曲线：由用户上传或选择，作为用户侧 profile 使用，不默认绑定地区。
- 中长期/日前持仓：属于交易策略，按 `profile` 管理，不按地区管理。
- 全局模型默认参数：如储能默认效率、投资单价、财务默认值，除非后续明确需要地区差异。

## 数据可信度记录

当前不追求公开完整来源信息，也不要求记录爬取链接、接口地址等敏感来源。只需要可内部追踪“数据是否可信”。

建议新增一个轻量可信度登记文件：

```text
data/catalog/data_trust.csv
```

建议字段：

```text
data_type,file_path,region,date_start,date_end,trust_level,is_mock,notes
```

字段解释：

- `data_type`：如 `spot_price`、`administrative_tariff`、`pv_curve`、`load_profile`。
- `file_path`：相对项目根目录的数据文件路径。
- `region`：无地区维度的数据填 `global` 或空。
- `date_start/date_end`：适用日期范围；典型曲线可留空或写 `typical`。
- `trust_level`：建议枚举 `mock`、`web_collected`、`official_api`、`manual_verified`。
- `is_mock`：`true/false`，方便程序过滤 mock 数据。
- `notes`：内部备注，不写敏感来源。

## 当前采纳的数据类型

- 行政分时电价：保留，按省份/月/用电类别。
- 合同分时电价：保留，按 profile 或后续业务合同模板。
- 现货价格：保留日前统一出清价、实时统一出清价；不做节点价、分区价、辅助服务等扩展。
- 用户负荷曲线：保留 16 种典型曲线，不按地区拆。
- 光伏出力：短期替换为真实数据，按省份和曲线性质组织。
- 中长期合约/日前持仓：继续保留，作为交易策略 profile，用于售电购电成本简化结算。
- 需量/容量单价：需要加入后端数据与接口，前端入口后续决定。

## 当前不采纳或暂缓的数据类型

- 天气数据：暂拿不到，不实现。
- 节点电价/物理节点价：当前业务无帮助，不实现。
- 输配电价、基金附加、系统运行费：可能三期实现，当前暂不考虑。
- 完整省级电力市场规则 YAML：当前按现有简化逻辑做，不扩展复杂规则库。
- 设备可靠性、故障率、衰减曲线、成本数据库等：当前暂不采纳，后续如进入真实运行仿真阶段再讨论。

## 已清理的现有不匹配项

以下偏复杂或不符合现阶段业务口径的字段/抽象，已从当前计算口径中移除；历史研究文档可保留，但不作为当前实现依据：

- `c_lt_block_yuan`：中长期合约阻塞附加费，当前业务口径不采用；读取层兼容旧列并丢弃。
- `C_lt_block`：结算结果中的阻塞分项，当前业务口径不采用。
- `PriceNode.PHYSICAL_NODE` / `price_node`：节点电价抽象，当前业务口径不采用。
- 文档中关于阻塞、节点、分摊、返还、复杂省级结算规则的内容需要标记为历史研究材料，不作为当前实现口径。
